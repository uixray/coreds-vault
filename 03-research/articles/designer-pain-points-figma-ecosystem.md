## Главные боли российских дизайнеров

**1. Рутина и выгорание (Routine & Burnout)** Переименование слоёв, выравнивание блоков, заполнение карточек «рыбой», ручная проверка отступов, цветов и шрифтов при дизайн-ревью — дизайнеры чаще всего жалуются на выгорание именно из-за рутины, тратя часы на переименование рабочих областей или перетаскивание блоков.

**2. Дизайн-ревью и хендофф разработчикам (Design Review & Handoff)** Каждому дизайнеру знакома картина: релиз, открываешь макеты в Figma, берёшь скриншот из прода и начинаешь проверять цвета, отступы, размеры шрифтов — это съедает очень много времени. Потерянные комментарии, задачи в пяти трекерах, разработчики неправильно понимают переходы между экранами.

**3. Дизайн-системы: создание и поддержка (Design System Maintenance)** Недостаточная документация — без чётких инструкций система бесполезна. Компоненты расползаются, нет единых стандартов именования, документация устаревает. Особенно остро у команд, где дизайнер один (как ты).

**4. Локализация инструментов (Localization Gap)** Большинство плагинов — англоязычные. Проверка орфографии русского языка, генерация «рыбы» на русском, работа с кириллической типографикой — всё это через костыли. Плагин Fish Text in Russian говорит сам за себя — он отвечает за генерацию случайного текста на русском языке, но это буквально единичный пример.

**5. Санкционные риски и оплата (Sanctions & Access)** Для российских пользователей в Figma доступны только личные аккаунты, корпоративные планы под вопросом, оплата через обходные пути. Это создаёт тревогу и потребность в инструментах, не зависящих от западной инфраструктуры.

**6. ИИ-инструменты не адаптированы под русский контекст** Существующие AI-плагины (FigPilot, Banani, Magician) работают через OpenAI/западные API — нейросети пока недостаточно развиты для профессиональных задач, не дают точных и предсказуемых результатов в контексте сложных задач, а для русского языка результаты ещё хуже.

## Анализ идеи «коллекция плагинов с Яндекс ИИ»

**Плюсы:**

- YandexGPT 5.1 Pro — лучшая модель Яндекса для обработки текстов, достигает уровня GPT-4.1. Русский язык — родной для модели.
- Есть YandexART для генерации изображений, OpenAI-совместимый API.
- Маркетинговый угол: «российские AI-технологии для российских дизайнеров» — патриотический нарратив, который хорошо резонирует.

**Минусы и риски:**

- Публичный API ограничивает один запрос 7 400 токенами, максимальный размер ответа — 2 000 токенов. Для серьёзных задач (анализ дизайн-систем, генерация документации) этого может не хватить.
- Привязка к Yandex Cloud — нужен аккаунт, оплата, API-ключи. Это создаёт барьер входа для пользователей.
- Ты становишься зависим от одного вендора. Если Яндекс изменит тарифы или API — вся экосистема под угрозой.
- Конкуренция: кто-то уже делает OpenAI-to-YandexGPT прокси (есть на GitHub).

## Мой совет: стратегия шире, чем «плагины с Яндекс ИИ»

Вместо привязки к одному AI-провайдеру, я бы предложил строить **DesignOps-экосистему «RayKit»** (или другое название) — набор инструментов и контента, решающих конкретные боли. AI — это один из слоёв, но не единственный.

### Tier 1 — Бесплатные плагины для Figma (рост аудитории)

Решают конкретные боли без AI, низкий барьер входа:

- **RuLint** — валидатор дизайн-системы: проверка именования слоёв, консистентности отступов, использования стилей/переменных. По сути — линтер для Figma-файлов по настраиваемым правилам.
- **RuText** — продвинутая генерация текста на русском: ФИО, адреса, телефоны, карточки товаров, отзывы — не Lorem Ipsum, а реалистичные русскоязычные данные.
- **HandoffKit** — экспорт спецификаций в Markdown/Obsidian-формат с автоматическим описанием компонентов, отступов, цветов. Мост между Figma и документацией.

### Tier 2 — AI-плагины (монетизация через freemium)

Здесь AI, но **провайдер-агностичный** подход (пользователь сам подключает YandexGPT, OpenAI, или любой OpenAI-совместимый API):

- **DesignReview AI** — сравнение макета со скриншотом продакшена, поиск расхождений. То, что тот дизайнер с Хабра пытался собрать за 4 недели — но в полированном виде.
- **DocGen AI** — автогенерация документации для компонентов дизайн-системы: описания, правила использования, do/don't.
- **CopyAssist** — генерация UX-текстов на русском прямо в Figma: кнопки, ошибки, онбординг, пустые состояния. С учётом контекста экрана.

### Tier 3 — Контент и комьюнити (личный бренд)

- **Telegram-канал** по DesignOps с фокусом на автоматизацию — кейсы, гайды, шаблоны. Это быстрее всего растит аудиторию в русскоязычном дизайн-комьюнити.
- **Obsidian-шаблоны** для DesignOps-документации — база знаний дизайн-команды «из коробки».
- **Открытые Figma-шаблоны** со встроенными правилами валидации.
- **Статьи на Хабр/vc.ru** с разбором процесса создания плагинов — это резонирует с аудиторией (статья про AI-плагин для ревью набрала хороший отклик).

## Почему эта стратегия лучше, чем «только Яндекс ИИ»

|Аспект|Только Яндекс AI|DesignOps-экосистема|
|---|---|---|
|Охват аудитории|Только те, кому нужен AI|Все дизайнеры с болями|
|Барьер входа|Нужен Yandex Cloud аккаунт|Бесплатные плагины = вирусный рост|
|Вендор-лок|Полный|Провайдер-агностичный|
|Личный бренд|«Человек, делающий обёртки над API»|«DesignOps-евангелист с инструментами»|
|Монетизация|Зависит от маржи на API|Freemium + контент + консалтинг|

## Что делать дальше

1. **Начать с RuLint или RuText** — простой плагин без AI, который решает очевидную боль и быстро набирает установки. Это твоя точка входа в Figma Community.
2. **Параллельно запустить Telegram-канал** о DesignOps-автоматизации — писать о процессе создания плагинов, делиться находками.
3. **Потом добавить AI-слой** — уже на базе существующей аудитории, с провайдер-агностичным подходом (YandexGPT как один из вариантов, но не единственный).
4. **Определиться с названием экосистемы** — единый бренд для всех инструментов.
