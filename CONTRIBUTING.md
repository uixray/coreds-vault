# Как контрибьютить в CoreDS Vault

Спасибо за интерес к проекту! Вот как вы можете помочь.

## Что можно делать

- **Исправлять ошибки** в существующих паттернах и документации
- **Добавлять примеры** из реальных продуктов в паттерны
- **Предлагать новые паттерны** через Issues
- **Обновлять устаревшие** заметки (см. freshness report)
- **Переводить** контент на другие языки
- **Улучшать скрипты** автоматизации

## Правила оформления

Перед контрибьюцией прочитайте [[00-meta/conventions|Соглашения по оформлению]].

Ключевые моменты:
- Каждый `.md` файл начинается с YAML frontmatter
- Используйте wikilinks `[[page]]` для внутренних ссылок
- Теги иерархические: `#type/pattern`, `#platform/ios`
- Status lifecycle: `seed → draft → review → stable → deprecated`

## Процесс

1. Fork репозитория
2. Создайте ветку: `git checkout -b feature/add-pattern-name`
3. Внесите изменения
4. Проверьте: frontmatter корректен, ссылки не битые
5. Создайте Pull Request с описанием изменений

## Вопросы

Создайте Issue в репозитории или свяжитесь с автором.
