# Генерация изображений и векторизация для Figma-плагина: полный обзор провайдеров и моделей

**Для Figma-плагина с поддержкой txt2img, img2img, inpainting, outpainting и векторизации (vectorization) оптимальная стратегия — гибридная архитектура.** Ни один облачный провайдер не покрывает все 5 возможностей полноценно. Лучший подход: облачные API для генерации (Replicate, Fal.ai или BFL дают наибольший охват при бюджете $20/мес) + **Recraft AI** как единственный провайдер с нативной генерацией SVG + клиентская векторизация через **VTracer WASM** прямо в iframe плагина. Для локального развёртывания на iGPU Radeon 780M реалистичен только **Stable Diffusion 1.5** через ONNX+DirectML; все остальные модели требуют выделенного GPU.

## Детальный разбор ключевых облачных провайдеров

### Stability AI — лидер по функциям редактирования (editing)

REST API доступен по адресу `https://api.stability.ai/v2beta/`. Предлагает модели **Stable Image Ultra** (SD 3.5 Large, $0.08/изобр.), **Stable Image Core** ($0.03/изобр.) и **SDXL 1.0** ($0.02). Главное преимущество — выделенные API-эндпоинты для inpainting ($0.05), erase, search-and-replace, а также Control-эндпоинты (Sketch, Structure, Style) для img2img ($0.05 каждый). **Outpainting** напрямую не поддерживается — доступен через Clipdrop Uncrop. Документация хорошая, есть OpenAPI-спецификация. CORS отсутствует — нужен прокси-сервер.

### OpenAI (gpt-image-1 / DALL-E) — лучшее качество по инструкциям

Новейшая модель **gpt-image-1** принимает изображения на вход и редактирует по текстовой инструкции. Ценовой диапазон широкий: от **$0.005** (gpt-image-1-mini, low quality) до **$0.17** (gpt-image-1, high quality) за 1024×1024. DALL-E 2 по-прежнему поддерживает классический endpoint для inpainting с маской. **$5 бесплатных кредитов** при регистрации — это ~1000 изображений на mini low. CORS **заблокирован** — прямые вызовы из браузера невозможны, обязателен серверный прокси.

### Replicate — максимальный выбор моделей

Маркетплейс с **50 000+ моделями**, включая все семейства FLUX, SD, Ideogram, Google Imagen и ключевой для проекта **Recraft V3 SVG**. Цены: FLUX.1 Schnell — **$0.003**/изобр., FLUX.1 Dev — $0.025, FLUX 1.1 Pro — $0.04. Community-модели тарифицируются по GPU-секундам ($0.000225–$0.001150/сек). Поддерживает все 5 возможностей, если считать модели для outpainting и Recraft для SVG. Документация превосходная с интерактивным тестированием моделей. Для CORS есть cors-proxy, но рекомендуется серверный прокси.

### Fal.ai — лучший для браузерной интеграции

**600+ моделей**, включая FLUX, SD, Recraft V3 SVG. Цены аналогичны Replicate: FLUX.1 Dev — $0.025, FLUX Kontext Pro — $0.04. Главное преимущество для Figma-плагина: **JS-клиент `@fal-ai/client` работает в браузерной среде**, что делает fal.ai наиболее дружелюбным к клиентским вызовам из iframe плагина. Поддерживает синхронные и асинхронные (queue-based) запросы.

### Black Forest Labs (FLUX) — прямой доступ к моделям FLUX

Оригинальный разработчик FLUX. API по адресу `https://api.bfl.ai`. Кредитная система: **1 кредит = $0.01**. Новейшие модели FLUX.2: Klein 4B от $0.014/изобр., Klein 9B от $0.015, Pro от $0.03, Flex от $0.05. FLUX.1 Fill — выделенная модель для inpainting ($0.05). FLUX.1 Kontext — для img2img ($0.04–$0.08). Документация отличная (Mintlify). Бесплатного уровня нет.

### Recraft AI ⭐ — единственный провайдер с нативной генерацией SVG

**Ключевой провайдер для задачи векторизации.** Recraft V3 генерирует SVG напрямую из текстового промпта за **$0.08/изобр.** (модель 20B — $0.044/изобр.). Отдельный эндпоинт для конвертации растра в SVG — **$0.01/изображение**. Также поддерживает все остальные возможности: txt2img (растр $0.04), img2img, inpainting, outpainting. Экспорт в SVG, PNG, JPG, **Lottie**, PDF. Модель Recraft V3 занимает **#1 на HuggingFace Text-to-Image Leaderboard** (ELO 1172). Доступен также через Replicate и fal.ai.

### Together AI — бесплатный FLUX.1 Schnell

Уникальное предложение: **полностью бесплатный endpoint** для FLUX.1 Schnell (rate-limited: 0.6 req/мин без карты, 3 req/мин с картой). OpenAI-совместимый API (`/v1/images/generations`). Поддерживает 25+ моделей включая FLUX 2.0, Imagen 4, Seedream. Однако inpainting и outpainting поддержаны слабо. **$1 бесплатных кредитов** при регистрации.

### Бюджетная оценка при $20/месяц

При бюджете $20/мес на API можно получить примерно следующее количество генераций:

- **NovitaAI** (SD): ~13 300 изображений ($0.0015/шт)
- **Together AI** (Schnell free): **неограниченно** при rate limit
- **Fireworks** (Schnell): ~14 000 ($0.0014/шт)
- **Replicate** (Schnell): ~6 600 ($0.003/шт)
- **Fal.ai** (FLUX Dev): ~800 ($0.025/шт)
- **Recraft** (SVG V3): ~250 SVG ($0.08/шт) или 2000 конвертаций ($0.01/шт)
- **OpenAI** (mini low): ~4 000 ($0.005/шт)

## Локальные модели и бэкенды для самостоятельного развёртывания (self-hosted)

### Сценарий А: Локальная машина (AMD Ryzen 7 8845HS, Radeon 780M iGPU, 16 ГБ RAM)

Radeon 780M — это интегрированный GPU архитектуры RDNA3 (gfx1103) с **12 CU** и разделяемыми 16 ГБ RAM. Пропускная способность памяти — ~51 ГБ/с (DDR5 dual-channel), что в **18 раз ниже** RTX 3090 (936 ГБ/с). Реалистично для GPU можно выделить 8–12 ГБ через BIOS UMA Frame Buffer Size, оставляя 4–8 ГБ на систему.

| Модель | VRAM (FP16) | VRAM (квант.) | Работает на iGPU? | ONNX/DirectML | Лицензия |
|---|---|---|---|---|---|
| **SD 1.5** ⭐ | 4–6 ГБ | — | **✅ Да** | ✅ Отлично | CreativeML Open RAIL-M (коммерч. ✅) |
| **SD 2.1** | 4–6 ГБ | — | **✅ Да** | ✅ Хорошо | CreativeML Open RAIL++-M (коммерч. ✅) |
| **SDXL** | 8–12 ГБ | — | ⚠️ Маргинально | ⚠️ Частично | CreativeML Open RAIL++-M (коммерч. ✅) |
| **SD 3.5 Medium** | 5–6 ГБ (FP8) | — | ❌ Нет пути | ❌ | Community (<$1M бесплатно) |
| **SD 3.5 Large** | 18 ГБ | 11 ГБ (FP8) | ❌ Нет | ❌ | Community (<$1M бесплатно) |
| **FLUX.1 Schnell** | 24 ГБ | 6–8 ГБ (Q4 GGUF) | ❌ Нет | ❌ | Apache 2.0 (коммерч. ✅) |
| **FLUX.1 Dev** | 24 ГБ | 6–8 ГБ (Q4 GGUF) | ❌ Нет | ❌ | Non-commercial ⚠️ |

| **PixArt-α** | 8–14 ГБ | 6–7 ГБ (4-bit) | ⚠️ Только CPU | ❌ | Research only ⚠️ |

**Единственный реалистичный вариант для Radeon 780M — Stable Diffusion 1.5** через ONNX Runtime + DirectML + Microsoft Olive для оптимизации. Готовые ONNX FP16-модели доступны на GitHub (Amblyopius/Stable-Diffusion-ONNX-FP16). При 512×512 ожидайте **30–120 секунд на изображение**. Применение **LCM LoRA** (Latent Consistency Model) сократит количество шагов с 20–30 до 4–8, пропорционально ускоряя генерацию.

Альтернативный путь — **ZLUDA** (неофициальные ROCm-библиотеки для gfx1103, репозиторий likelovewant/ROCmLibs-for-gfx1103-AMD780M-APU). По отзывам, даёт ускорение в 2–3 раза по сравнению с DirectML, но менее стабилен и сложнее в настройке.

**SDXL** теоретически запускается на 512×512 с агрессивными оптимизациями (lowvram, attention slicing, VAE tiling), но будет крайне медленным и нестабильным. **Не рекомендуется.** FLUX на DirectML **не работает** из-за жёстко закодированных CUDA-операций и неподдерживаемых операций float8.

**Возможности SD 1.5 на iGPU:** txt2img ✅, img2img ✅, inpainting ✅ (через специализированную модель SD 1.5 Inpainting), outpainting ✅ (через ControlNet/community-модели). Качество — уровня 2022 года (★★★☆☆), но есть огромная экосистема LoRA-моделей (DreamShaper, Realistic Vision и др.).

### Бэкенды для инференса (inference backends)

**ComfyUI** ⭐ — лучший выбор. Поддерживает все модели (SD 1.x–3.x, SDXL, FLUX, PixArt, Kandinsky через кастомные ноды). Полноценный REST API на порту 8188 (`POST /prompt` с JSON-воркфлоу). DirectML через `--directml` флаг, но с ограничениями: FLUX на DirectML не работает, поддержка памяти ограничена. GGUF-квантизация через ComfyUI-GGUF plugin. Для удалённого GPU-сервера — безусловный лидер.

**AUTOMATIC1111 WebUI** — зрелый проект с хорошо документированным REST API (флаг `--api`). AMD-форк lshqqytiger/stable-diffusion-webui-amdgpu поддерживает DirectML и ZLUDA. Интеграция Microsoft Olive для ONNX-оптимизации. Лучше протестирован с DirectML, чем ComfyUI.

**Forge** (stable-diffusion-webui-forge) — форк A1111 с существенной оптимизацией памяти. Нативная поддержка FLUX NF4/GGUF с LoRA. AMD-форк: lshqqytiger/stable-diffusion-webui-amdgpu-forge. Лучший вариант в стиле A1111 для сценариев с малым VRAM. Оригинальный проект lllyasviel больше не обновляется (с июля 2025).

**InvokeAI** — профессиональный UI с холстом для inpainting/outpainting. REST API на порту 9090. **Не поддерживает DirectML.** AMD только через ROCm на Linux, с частыми проблемами. Не рекомендуется для AMD iGPU, но хорош для удалённого NVIDIA-сервера.

**ONNX Runtime + DirectML + Microsoft Olive** — единственный надёжный путь для GPU-ускорения на Radeon 780M. Olive конвертирует PyTorch → ONNX, применяет графовые оптимизации и квантизацию FP16. Для SD 1.5 задокументировано ускорение до **9.9× на AMD Radeon** по сравнению с PyTorch по умолчанию. Готовые ONNX-версии есть для SD 1.5 и SD 2.1; для SDXL — возможна конвертация, но сложнее. Для SD3, FLUX, Kandinsky, PixArt — **ONNX-версий нет**.

## Векторизация (vectorization): инструменты и подходы

### Инструменты, работающие прямо в Figma-плагине

Архитектура Figma-плагина: **код плагина** работает в QuickJS (JS-VM, скомпилированная в WASM) — **не поддерживает загрузку сторонних WASM-модулей**. **UI плагина** работает в iframe с полноценными браузерными возможностями, включая WebAssembly. Стратегия: запускать векторизацию в iframe, передавать результат в код плагина через `postMessage`.

**imagetracerjs** ⭐ — **самый простой вариант для интеграции**. Чистый JavaScript, работает и в коде плагина (QuickJS), и в iframe. API: `ImageTracer.imagedataToSVG(imgd, options)` — синхронный, принимает стандартный `ImageData`. Множество пресетов: posterized, curvy, sharp, detailed, artistic. Лицензия: **Public Domain (Unlicense)** — никаких ограничений. Качество среднее — подходит для простых график и иллюстраций, но слабо на фотореалистичных изображениях. Уже используется в существующих Figma-плагинах (Free Vectorizer, Image Tracer).

**VTracer (visioncortex/vtracer)** ⭐ — **лучший баланс качества и совместимости**. Алгоритм O(n) (против O(n²) у Potrace), полноцветная кластеризация, компактный SVG-вывод. Rust-реализация с **WASM-портом** — NPM-пакет `vectortracer` (AlansCodeLog/vectortracer). Работает в iframe через Web Worker. Принимает `ImageData` из JavaScript. Лицензия: **MIT OR Apache-2.0** — идеально для коммерческого плагина. Более компактный SVG-вывод, чем Adobe Illustrator Image Trace. Уже применяется в Figma-плагинах.

**Potrace (WASM)** — стандарт индустрии для монохромной векторизации. WASM-порты: `esm-potrace-wasm` (ESM-сборка), `potrace-wasm`. Используется в Figma-плагине "Tracer". Отличное качество для чёрно-белого (line art, логотипы). **⚠️ Лицензия GPL-2.0** — вирусный копилефт, плагин должен быть GPL-лицензирован при распространении. **Работает только с монохромом** — для цвета нужны множественные проходы.

### AI-модели для векторизации (серверное развёртывание)

**LIVE (Layer-wise Image Vectorization)** — CVPR 2022 Oral. Послойная генерация SVG через рекурсивное добавление оптимизируемых кривых Безье. Высокое качество для простых и средних изображений, компактный SVG с семантической структурой слоёв. **~5 часов** на изображение 2K. Требует Python, PyTorch, CUDA. Лицензия Apache-2.0.

**SVGDreamer** — CVPR 2024. Генерация SVG из текстового промпта через Stable Diffusion + DiffVG. Стили: иконография, скетч, пиксель-арт, лоу-поли, живопись. Качество выше VectorFusion и DiffSketcher. Лицензия MIT. Только серверная инференция с GPU.

**StarVector** — мультимодальная LLM, генерирующая SVG-код из изображений или текста. Компактные SVG (~3k токенов vs ~18k у LIVE). По человеческим оценкам превосходит LIVE, AutoTrace и VTracer для сложных изображений. Требует LLM-сервер с GPU.

**DiffVG** — дифференцируемый растеризатор SVG (SIGGRAPH Asia 2020). Не векторизатор сам по себе, а инфраструктурный компонент для LIVE, SVGDreamer и др. Лицензия Apache-2.0.

**LayerPeeler** — SIGGRAPH Asia 2025. Авторегрессивная послойная векторизация с VLM для анализа перекрытий. Превосходит LIVE для сложных изображений с окклюзиями. Серверная инференция.

### Облачные API для векторизации

**Recraft AI** ⭐⭐ — **лучший облачный вариант для проекта**. Два режима:
- Нативная генерация SVG из текста: **$0.08/изобр.** (V3) или **$0.044** (20B)
- Конвертация растра в SVG: **$0.01/изображение** — очень доступно
- Экспорт: SVG, PNG, JPG, Lottie, PDF
- Также доступен через Replicate ($0.04/изобр.) и fal.ai

**Vectorizer.AI** — специализированный сервис конвертации растр→вектор. AI + классические алгоритмы, распознавание геометрических фигур, очистка краёв. API: `POST /api/v1/vectorize`. Цена: **$0.175–0.20/конвертация** (дорого). Вход: JPEG, PNG, WEBP, BMP, GIF (макс. 2 МП). Выход: SVG, PDF, EPS, DXF. HTTP Basic Auth. Тестовые вызовы бесплатны. Тот же разработчик, что и Vector Magic.

**Vector Magic** — одна из лучших коммерческих программ-векторизаторов. Онлайн-версия: $5.49/мес. **Публичного API нет** — только ручной интерфейс. Не подходит для автоматической интеграции.

**Adobe Illustrator Image Trace** — индустриальный стандарт. **API недоступен.** Функция доступна только в десктопном приложении Illustrator. Adobe Firefly API поддерживает генерацию изображений, но не трассировку.

### Классические инструменты

**AutoTrace** — альтернатива Potrace с поддержкой цвета. GNU GPL. Последний релиз: v1.16 (сентябрь 2019) — **фактически заброшен**. Нет JS/WASM-портов. Только серверное развёртывание.

## Практическая архитектура Figma-плагина

### CORS и сетевой доступ

Большинство API-провайдеров **блокируют CORS** для прямых браузерных вызовов. Два решения:

- **Серверный прокси** — лёгкий Cloudflare Worker или Vercel Edge Function, проксирующий запросы к API. Наиболее безопасный подход (API-ключи не попадают на клиент). Минимальная стоимость: бесплатные уровни Cloudflare Workers (100 000 запросов/день) или Vercel Edge (100 000 вызовов/мес).
- **`networkAccess` в manifest.json** — Figma-плагины могут декларировать разрешённые домены для HTTP-запросов. Работает для API с CORS-заголовками (**fal.ai**, **Hugging Face Inference API**).

### Рекомендуемая трёхуровневая архитектура

**Уровень 1 — Клиентская векторизация (бесплатно, мгновенно, оффлайн):**
VTracer WASM в iframe для высококачественной цветной векторизации + imagetracerjs как лёгкий fallback. Обе библиотеки совместимы с коммерческим использованием (MIT, Public Domain).

**Уровень 2 — Облачные API (качество, $20/мес бюджет):**
- **Генерация:** Together AI (FLUX Schnell, бесплатно) для быстрых превью + Fal.ai или Replicate (FLUX Dev/Pro, $0.025–0.04) для финальных изображений
- **Inpainting/Editing:** Stability AI (выделенные endpoints, $0.05) или OpenAI gpt-image-1 (инструктивное редактирование)
- **Outpainting:** Clipdrop Uncrop ($0.003–0.04/кредит) или FLUX.1 Fill через BFL API ($0.05)
- **SVG-генерация:** Recraft AI ($0.08/SVG или $0.01/конвертация)

**Уровень 3 — Self-hosted для продвинутых пользователей:**
- Локально: SD 1.5 + ONNX + DirectML + Olive (бесплатно, 30–120 сек/изобр.)
- Удалённо: ComfyUI на RunPod RTX 3090 ($0.44/час) с FLUX Schnell GGUF Q8

## Ключевые выводы и рекомендации

Ландшафт API-провайдеров генерации изображений в 2026 году чрезвычайно конкурентен. Цены на базовую генерацию txt2img упали до **$0.001–0.003** за изображение (NovitaAI SD, Fireworks Schnell), что делает облачные API доступнее локального развёртывания для большинства сценариев. Однако **векторизация остаётся узким местом**: только Recraft AI предлагает нативную генерацию SVG, а специализированный Vectorizer.AI берёт **$0.175/конвертация** — в 50 раз дороже генерации растра.

Для проекта Figma-плагина критически важны три технических решения. Во-первых, **серверный прокси обязателен** — из 15 исследованных провайдеров только fal.ai и Hugging Face надёжно поддерживают CORS. Cloudflare Worker бесплатного уровня полностью решает эту проблему. Во-вторых, **клиентская векторизация через VTracer WASM** — это killer feature: мгновенная, бесплатная, оффлайн-доступная конвертация прямо в плагине, без серверных вызовов. В-третьих, на Radeon 780M iGPU реалистичен **только SD 1.5** — все модели нового поколения (FLUX, SD3, SDXL) требуют выделенного GPU минимум с 8–12 ГБ VRAM.

Наиболее перспективная стратегия — начать с бесплатного FLUX Schnell (Together AI) + VTracer WASM для MVP, затем расширять набор провайдеров по мере роста пользовательской базы, добавляя Recraft для SVG и Stability AI для продвинутого редактирования.
