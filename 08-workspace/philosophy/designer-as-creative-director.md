# Дизайнер как Творческий Директор

> *Настоящая работа дизайнера — не исполнение, а режиссура.*

---

## Два режима работы

Любой дизайнер постоянно переключается между двумя принципиально разными режимами:

### Designer 1.0 — Исполнитель
Рутинные операции, которые отнимают время но не требуют творческого мышления:

- Переименовать 200 слоёв в Figma по конвенции BEM
- Вставить реалистичные тексты-заполнители в макет
- Проверить что все цвета привязаны к переменным
- Найти дубли одинаковых иконок на странице
- Сформатировать тире и кавычки в UX-копи
- Прочитать 30 постов из профессиональных каналов и вычленить главное
- Искать нужные файлы через поиск в папках

### Designer 2.0 — Режиссёр
Работа, которую не может сделать машина:

- Понять настоящую проблему пользователя
- Спроектировать информационную архитектуру
- Принять решение: один экран или два?
- Оценить качество результата и дать направление
- Придумать метафору, которая делает продукт понятным
- Сформулировать принципы компонентной системы
- Объяснить команде **почему** именно так

**Задача автоматизации:** перевести Designer 1.0 → на инструменты, освободив максимум времени для Designer 2.0.

## Инструментальный стек

Стек организован по слоям — от уровня операционной системы до специализированных сервисов:

```
┌────────────────────────────────────────────────────────────┐
│  SYSTEM LAYER  — DesignOps Assistant (AutoHotkey)          │
│  AI везде: в любом приложении, без переключения контекста  │
│  Типографика · AI Spotlight · Сниппеты · Deep Links        │
├────────────────────────────────────────────────────────────┤
│  FIGMA LAYER  — Плагины для рабочего процесса              │
│  UText (тексты) · URenaming (слои) · UData (данные)        │
│  Design Lint (аудит) · UVectorFinder (дубли)               │
├────────────────────────────────────────────────────────────┤
│  DATA LAYER  — Информация сама приходит к тебе             │
│  tg-digest-bot (дайджесты каналов) · AI Design Ops         │
│  (мульти-агентная исследовательская система)               │
├────────────────────────────────────────────────────────────┤
│  INFRA LAYER  — Инфраструктура, которую не видно           │
│  figma-ai-proxy (единый прокси для всех плагинов)          │
│  LM Studio (локальные модели без API-ключей)               │
└────────────────────────────────────────────────────────────┘
```

### System Layer: DesignOps Assistant

[[08-workspace/system/designops-assistant|DesignOps Assistant]] — AutoHotkey-приложение для Windows, которое добавляет AI-ассистента и профессиональную типографику **на уровне всей операционной системы**.

Выдели текст в любом приложении → `Alt+J` → выбери команду → AI отвечает прямо в поле ввода.
Контекстная адаптация: в Figma — краткий UX-текст без Markdown, в Obsidian — структурированная заметка, в VS Code — чистый код.

### Figma Layer: Плагины

| Плагин | Что автоматизирует |
|--------|-------------------|
| [[08-workspace/figma-plugins/utext\|UText]] | Генерация и применение текста через AI, 30+ моделей |
| [[08-workspace/figma-plugins/urenaing\|URenaming]] | Массовое переименование слоёв (BEM, camelCase, AI-режим) |
| [[08-workspace/figma-plugins/udata\|UData]] | Пресеты реалистичных данных для макетов |
| [[08-workspace/figma-plugins/design-lint\|Design Lint]] | Аудит: непривязанные цвета, шрифты, эффекты, overrides |
| [[08-workspace/figma-plugins/uvectorfinder\|UVectorFinder]] | Поиск геометрически идентичных векторов |

### Data Layer: Информация без ручного труда

[[08-workspace/services/tg-digest-bot|tg-digest-bot]] — Telegram-бот, который сам собирает посты из профессиональных каналов, суммаризирует AI и доставляет дайджест. Roadmap v1.2: прямая интеграция с Obsidian vault.

[[08-workspace/services/ai-design-ops|AI Design Ops Orchestrator]] — локальная мульти-агентная система на FastAPI. Orchestrator маршрутизирует задачи между специализированными агентами: Research Lead (Perplexity), Visual Lead (Gemini), Tech Lead.

### Infra Layer: Невидимая инфраструктура

[[08-workspace/services/figma-ai-proxy|figma-ai-proxy]] — CORS-прокси, который позволяет Figma-плагинам обращаться к любым AI-провайдерам. Развёрнут на production-сервере (`proxy.uixray.tech`). Без него большинство плагинов не могут работать с облачными AI API из-за ограничений браузерной безопасности.

## Принципы автоматизации

### 1. Automation First
Если действие повторяется более трёх раз — создай инструмент. Время на создание инструмента окупается за несколько дней использования.

### 2. AI as Copilot, not Autopilot
AI не принимает решения — он выполняет механическую работу под твоим руководством. Ты задаёшь направление, оцениваешь результат, принимаешь финальное решение.

### 3. Context-Aware Tools
Инструменты должны адаптироваться к контексту работы. DesignOps Assistant знает, что ты в Figma — и отвечает короче, без Markdown. UText знает, что слой называется `user_name` — и генерирует подходящие имена.

### 4. Single Source of Truth
- Все знания → в vault (Obsidian)
- Все токены → в DTCG JSON (`05-tokens/`)
- Все паттерны → в Pattern Library (`02-patterns/`)
- Всё остальное → ссылки и связи

### 5. Zero Context Switch
Лучший инструмент — тот, что не требует переключения. `Alt+J` прямо в Figma. Дайджест в Telegram. Аудит одной кнопкой внутри плагина.

## Vault как среда обитания дизайнера

Этот vault — не только справочник по компонентам. Это **интегрированная среда**, где:

- Знания о паттернах → связаны с компонентами через `related_patterns`
- Исследования → питают паттерны и ADR
- Токены → генерируют код для всех платформ
- Инструменты → документированы здесь, в `08-workspace/`
- Автоматизация → сканирует свежесть, проверяет связи

Дизайнер в этой системе занимается тем, чем должен заниматься — **проектирует**, а не администрирует.
