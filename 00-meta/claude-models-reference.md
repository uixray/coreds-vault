Вот актуальная раскладка по моделям Claude на февраль 2026:

**Opus 4.6** (я сейчас) — самая мощная модель. Глубокий анализ, сложные рассуждения, длинный контекст до 200K (до 1M в API). Идеальна для архитектурных решений, ревью кода, комплексного анализа дизайн-систем, написания документации с нуля. Минус — медленнее остальных и расходует больше лимита сообщений.

**Sonnet 4.5** — рабочая лошадка. Лучшее соотношение скорости и интеллекта. Позиционируется Anthropic как лучшая модель для кода и агентов. Отлично подходит для ежедневных задач: генерация компонентов, работа с API, рефакторинг, фронтенд-разработка. Выдаёт чистый, рабочий код.

**Haiku 4.5** — самая быстрая и дешёвая. Доступна бесплатным пользователям. Хороша для быстрых вопросов, скаффолдинга UI, простых правок, автоматизации рутины. Но теряет контекст в длинных сессиях.

---

**Что выбирать под твои задачи:**

Для оценки дизайн-систем, написания гайдлайнов, сложной документации → **Opus 4.6** — глубже анализирует, лучше держит структуру большого документа.

Для повседневной работы с Figma MCP, генерации кода, TypeScript, автоматизаций → **Sonnet 4.5** — быстро, качественно, экономит лимит.

Для быстрых уточнений, простых скриптов, мелких правок → **Haiku 4.5** — моментальные ответы без лишнего.

**Что можно делать дальше:**

- Попробовать переключаться между моделями прямо в чате через селектор моделей вверху
- Настроить Claude Code с разными моделями для разных типов задач
- Использовать Sonnet как основную модель, а Opus подключать для ревью и сложного анализа

Вот практическое руководство по настройке моделей в Claude Code:

## 3 способа переключения (по приоритету)

**1. Прямо в сессии** — команда `/model`

```bash
/model sonnet    # переключиться на Sonnet 4.5
/model opus      # переключиться на Opus 4.6
/model haiku     # переключиться на Haiku 4.5
/model opusplan  # гибридный режим (см. ниже)
```

Работает мгновенно, без перезапуска. `/model` без аргументов покажет текущую модель и список доступных.

**2. При запуске сессии** — флаг `--model`

```bash
claude --model sonnet    # запустить сессию на Sonnet
claude --model opus      # запустить сессию на Opus
```

**3. Постоянный дефолт** — переменные окружения

Добавь в профиль PowerShell (`$PROFILE`) или системные переменные:

```powershell
# Основная модель по умолчанию
$env:ANTHROPIC_MODEL = "sonnet"

# Или закрепить конкретные версии для каждого тира
$env:ANTHROPIC_DEFAULT_OPUS_MODEL = "claude-opus-4-6"
$env:ANTHROPIC_DEFAULT_SONNET_MODEL = "claude-sonnet-4-5-20250929"
$env:ANTHROPIC_DEFAULT_HAIKU_MODEL = "claude-haiku-4-5-20251001"
```

Или через `settings.json` (`~/.claude/settings.json`):

```json
{
  "model": "sonnet"
}
```

## Opus Plan Mode — автоматическое переключение

Самая полезная фича для твоего воркфлоу:

```bash
/model opusplan
```

Работает так: **Opus** занимается планированием и архитектурой (plan mode), затем **автоматически переключается на Sonnet** для написания кода (execution mode). Получаешь глубокий анализ Opus + скорость Sonnet без ручного переключения.

---

## Effort Level — уровень усилий для Opus 4.6

Контролирует глубину рассуждений Opus — `low`, `medium`, `high` (по умолчанию):

```bash
# Через переменную окружения
$env:CLAUDE_CODE_EFFORT_LEVEL = "medium"

# Или в /model — стрелками влево/вправо регулируется ползунок
```

`low` — быстрые простые задачи, `high` — сложный анализ и рефакторинг.

## Рекомендуемый воркфлоу под твои задачи

**Sonnet как дефолт** → для ежедневной работы с TypeScript, AutoHotKey, автоматизаций

**`/model opus`** → когда нужно разобраться в архитектуре, написать документацию дизайн-системы, сложный рефакторинг

**`/model haiku`** → быстрые вопросы по синтаксису, мелкие правки, генерация шаблонов

**`/model opusplan`** → для крупных задач: Opus спланирует, Sonnet выполнит

**Полезные команды:**
- `/cost` — показать расход токенов за сессию
- `/status` — текущая модель и аккаунт
- `/clear` — очистить контекст перед новой задачей (экономит токены)

**Что можно делать дальше:**
- Настроить `$PROFILE` в PowerShell с дефолтной моделью и алиасами
- Попробовать `opusplan` на реальном проекте — например, рефакторинг дизайн-гайдлайнов
- Настроить MCP-серверы в `~/.claude/settings.json` для интеграции с Figma

**Cowork** — это по сути Claude Code, но без терминала и для нетехнических задач. Работает как десктопный AI-агент прямо в приложении Claude Desktop (уже доступен на Windows).

## Как это работает

Ты указываешь Claude папку на компьютере, описываешь задачу на естественном языке, и он сам планирует шаги, выполняет их, при необходимости запускает параллельные подзадачи. Ты видишь прогресс и можешь вмешаться в любой момент.

## Что умеет

Работа с файлами — сортировка, переименование, создание документов (docx, xlsx, pptx, pdf) прямо из данных в папке. Например, собрать отчёт по расходам из скриншотов чеков или превратить разрозненные заметки в структурированный документ.

Коннекторы (connectors) — подключение к внешним сервисам: Slack, Notion, GitHub, Google Drive, CRM и сотни других через MCP-протокол. Данные из сервисов можно сохранять локально и наоборот.

Интеграция с Claude in Chrome — если подключить, Cowork может выполнять задачи, требующие браузера.

Навыки (skills) — встроенная поддержка форматов xlsx, pptx, docx, pdf с возможностью создавать свои скиллы.

Глобальные и папочные инструкции (global/folder instructions) — можно задать тон, формат, контекст роли, и они будут применяться ко всем сессиям или только при работе в конкретной папке.

## Доступность

Работает на всех платных планах (Pro, Max, Team, Enterprise). Статус — research preview. Приложение должно оставаться открытым во время работы, память между сессиями не сохраняется.

## Чем отличается от обычного чата

Обычный Claude **показывает как**. Cowork **делает**. Он автономно выполняет многошаговые задачи с реальными файлами на твоём компьютере, а не просто генерирует текст в чате.
