#### A2.9* — Design Checklist: детализация

**Предустановленные чеклисты:**
- Создание дизайн-системы
- Этапы работы: Discovery → Research → Design → Testing → Handoff → Review → Support
- Создание компонентов: состояния, a11y, поведения, edge cases
- Популярные паттерны: авторизация, каталог, профиль, онбординг — что содержит, что опционально, что учесть
- Тёмная тема
- Переключение между модами девайсов (responsive, adaptive)

**Встроенные гайды:**
- Material Design 3
- iOS Human Interface Guidelines
- Web design best practices
- Эвристики Нильсена

**Функционал:**
- Каждый пункт сопровождается ссылками на материалы
- Пользователь может загружать свои чеклисты (JSON/Markdown)
- Прогресс-бар по каждому чеклисту
- Привязка чеклиста к конкретному файлу/проекту через pluginData
- (Продвинуто) AI-ассистент: проверка выбранного макета на соответствие гайду или чеклисту

## Заметки и принципы

### Разделение ботов
- **Дайджест-бот** (C2) — персональный ассистент: дайджесты, заметки, публикации
- **Design Review Bot** (C1) — командный: ревью макетов, сбор фидбека
- **Image Token Bot** (C3.1) — утилитарный, микро-бот
- **ImageGen Bot** (C3.2) — утилитарный, генерация картинок

### Архитектура интеграций
- **Telegram** — облачный API, работает из Figma Plugin без ограничений
- **Obsidian / Eagle** — localhost API, для dev mode напрямую, для продакшена через локальный прокси (E6)
- **Claude** — через MCP сервер или Figma MCP connector

### Подход к разработке
- MVP first — минимальная рабочая версия, потом итерации
- Единый boilerplate (E1) и UI kit (E2) для всех плагинов
- Open Source по умолчанию — GitHub uixray
- Документация на каждый плагин с первого дня
