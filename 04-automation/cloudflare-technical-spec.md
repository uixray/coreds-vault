Вот подробное Техническое Задание (ТЗ), которое вы можете скопировать и вставить в Claude Code (или другой ИИ-агент с доступом к терминалу), а также пошаговая инструкция для вас.

---

### Часть 1: Техническое задание для Claude Code

**Скопируйте этот текст и отправьте его Claude Code, когда будете находиться в терминале вашего сервера (или когда дадите ему доступ к проекту):**

> **Role:** Network Engineer / DevOps
> 
> **Objective:** Реализовать двухэтапную систему обхода блокировок:
> 
> 1. Настройка "Узла Выхода" (Зарубежный VPS) с Amnezia (XRay/VLESS) и Cloudflare Tunnel.
>     
> 2. Настройка "Транзитного Прокси" (Сервер Яндекс), перенаправляющего трафик на Узел Выхода.
>     
> 
> **Project 1: Foreign VPS (Exit Node)**
> 
> - Установить Docker и Docker Compose.
>     
> - Развернуть протокол **VLESS + Reality** (через компоненты Amnezia или XRay) для максимальной маскировки.
>     
> - Настроить **Cloudflare Tunnel (cloudflared)** для скрытия реального IP сервера. Весь входящий трафик должен идти через туннель CF.
>     
> - Сгенерировать клиентский конфиг (JSON или ссылку) для подключения со стороны Яндекс-сервера.
>     
> 
> **Project 2: Yandex Server (Relay Node)**
> 
> - Проанализировать текущий прокси на сервере Яндекс (если установлен 3proxy, Nginx или Dante).
>     
> - Настроить **Upstream Forwarding**: весь трафик, поступающий на этот прокси, должен пересылаться на Узел Выхода (Project 1) через созданный зашифрованный туннель.
>     
> - Реализовать схему: `User -> Yandex Proxy -> [Encrypted Tunnel] -> Foreign VPS -> Internet`.
>     
> 
> **Technical Requirements:**
> 
> - Использовать современные протоколы (VLESS-Reality или Shadowsocks-2022).
>     
> - Обеспечить автозапуск всех компонентов через systemd.
>     
> - Настроить базовый Firewall (ufw), открыв только необходимые порты.
>     
> 
> **Output:** После настройки предоставь финальные параметры подключения и статус работы служб.

### Часть 2: Инструкция по реализации для вас

Чтобы эта схема заработала, вам нужно выполнить следующие действия по порядку:

#### Шаг 1: Подготовка серверов

1. **Зарубежный VPS (Exit Node):** Купите любой дешевый VPS вне РФ (Нидерланды, Германия, Казахстан и т.д.). Операционная система — Ubuntu 22.04.
    
2. **Яндекс Сервер:** Убедитесь, что у вас есть SSH-доступ к нему.
    

#### Шаг 2: Настройка Cloudflare (для Project 1)

1. Зайдите в панель Cloudflare -> Zero Trust -> Networks -> Tunnels.
    
2. Создайте новый туннель (например, `exit-node-tunnel`).
    
3. Скопируйте команду установки (обычно начинается с `curl ... cloudflared service install ...`).
    
4. **Дайте эту команду Claude Code**, чтобы он интегрировал её в процесс настройки.
    

#### Шаг 3: Запуск Claude Code

1. Зайдите по SSH на **Зарубежный VPS**.
    
2. Запустите Claude Code в папке проекта.
    
3. Вставьте ТЗ (из Части 1 выше).
    
4. После того как Claude настроит сервер, он выдаст вам **Client Config** (ключ или файл). Сохраните его.
    

#### Шаг 4: Настройка связи (Яндекс -> VPS)

1. Зайдите по SSH на **Сервер Яндекс**.
    
2. Запустите Claude Code там.
    
3. Скажите ему: _"Вот конфиг от моего зарубежного сервера [Вставьте конфиг]. Настрой текущий прокси так, чтобы он использовал этот сервер как апстрим (перенаправлял весь трафик через него)"_.
    

---

### Почему это сработает?

1. **Локальные ограничения:** Провайдер видит ваш трафик до сервера Яндекса (это доверенный узел внутри страны) или до Cloudflare (это мировой гигант, который не блокируют полностью).
    
2. **Обход DPI:** Внутри туннеля используется протокол Amnezia (VLESS/Reality), который имитирует обычный просмотр сайта (HTTPS). Цензурные фильтры не видят в этом VPN-трафика.
    
3. **Двойной прыжок:** Ваша анонимность выше, так как конечный сайт видит IP зарубежного сервера, а ваш провайдер видит только соединение с Яндексом.
    

### Что может пойти не так (нюансы):

- **Скорость:** Из-за "двойного прыжка" (Double Hop) задержка (ping) может вырасти. Это нормально для обхода блокировок, но может быть критично для игр.
    
- **Cloudflare Tunnel:** Cloudflare может блокировать пропуск через себя определенных видов UDP-трафика (например, торрентов) на бесплатном тарифе. Для обычных сервисов и сайтов это работает идеально.
