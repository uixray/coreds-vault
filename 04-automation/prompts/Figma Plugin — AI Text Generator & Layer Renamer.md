# Figma Plugin: AI Text Generator & Layer Renamer

## Описание проекта
Создай плагин для Figma с интеграцией LLM-провайдеров (LM Studio, Яндекс Cloud, OpenAI-совместимые API) для генерации текста и умного переименования слоёв.

## Технический стек
- Figma Plugin API (TypeScript)
- UI: Preact или vanilla TypeScript (лёгкий бандл)
- Сборка: esbuild или Vite
- Стили: CSS Modules или Tailwind (inline)

## Критические технические ограничения Figma

### Сетевые запросы
- Plugin UI (iframe) НЕ может делать fetch напрямую к внешним API
- Решение: использовать `figma.ui.postMessage()` для связи UI ↔ Sandbox, а сетевые запросы делать через:
  - Вариант A: Внешний relay-сервер (рекомендуется для продакшена)
  - Вариант B: Для LM Studio на localhost — проксировать через локальный сервер с CORS-заголовками

### CORS для LM Studio
- LM Studio по умолчанию слушает `http://localhost:1234`
- Нужно включить CORS в настройках LM Studio или использовать прокси

### Хранение данных
- `figma.clientStorage` — для API-ключей, endpoint, пресетов (асинхронный, лимит ~1MB)
- Чувствительные данные (ключи) — предупреди пользователя о безопасности

## Фазы разработки

### ФАЗА 1: Базовая инфраструктура и LLM-интеграция (MVP)

#### 1.1 Структура плагина
```
/plugin
  /src
    /ui
      main.tsx          # Точка входа UI
      App.tsx           # Главный компонент
      /components
        ProviderSettings.tsx
        TextProcessor.tsx
        Preview.tsx
        TokenCounter.tsx
    /sandbox
      code.ts           # Figma sandbox (работа с документом)
      api-bridge.ts     # Мост для API-запросов
    /shared
      types.ts          # Общие типы
      constants.ts
    /lib
      llm-client.ts     # Универсальный клиент для LLM
      token-counter.ts  # Подсчёт токенов
  manifest.json
```

#### 1.2 Модуль подключения к провайдерам
- Форма настройки: endpoint URL, API-ключ, выбор модели
- Кнопка "Тест соединения"
- Сохранение в clientStorage
- Поддержка OpenAI-совместимого API формата

#### 1.3 Подсчёт токенов и стоимости
- Использовать библиотеку `tiktoken` (или упрощённую оценку: ~4 символа = 1 токен)
- Показывать перед отправкой:
  - Примерное количество входных токенов
  - Оценка стоимости (пользователь задаёт цену за 1K токенов в настройках)
- После ответа — показать фактическое использование (если API возвращает usage)

#### 1.4 Базовая генерация текста
- Выбор текстовых слоёв в Figma
- Поле ввода промпта
- Поле системного промпта (collapsible)
- Кнопка "Сгенерировать"
- **Streaming-вывод**: если API поддерживает — показывать текст посимвольно (typewriter effect)
- **Предпросмотр**: результат показывается в UI, пользователь подтверждает перед применением

#### 1.5 UI индикация процесса
- Состояния: idle → loading → streaming → preview → applied
- При streaming — анимация печати текста
- Прогресс-бар или спиннер для не-streaming запросов
- Кнопка отмены запроса

### ФАЗА 2: Пресеты и локальные данные

#### 2.1 Режим локальных данных (без LLM)
- Категории: имена, фамилии, адреса, телефоны, email, названия товаров, даты
- Дефолтные массивы данных (русские + английские варианты)
- Логика подстановки: случайный выбор или последовательный
- Импорт из Excel/CSV:
  - Парсинг через `xlsx` или `papaparse`
  - Маппинг колонок на категории
  - Сохранение в clientStorage

#### 2.2 LLM-пресеты
- Структура пресета:
```typescript
interface Preset {
  id: string;
  name: string;
  category: 'tone' | 'length' | 'typography' | 'custom';
  systemPrompt?: string;
  userPromptTemplate: string; // с плейсхолдером {{text}}
  isDefault: boolean;
}
```
- Дефолтные пресеты:
  - Тон: серьёзный, дружелюбный, весёлый, формальный
  - Длина: сократить, расширить, сделать лаконичным
  - Типографика: расставить кавычки, тире, убрать лишние пробелы
- CRUD для пользовательских пресетов
- Экспорт/импорт пресетов (JSON)

#### 2.3 Кнопка "Улучшить промпт"
- Отправляет промпт пользователя в LLM с мета-промптом:
  "Улучши и конкретизируй следующий промпт для генерации текста. Сделай его более точным и детальным: {{userPrompt}}"
- Результат показывается в том же поле, пользователь может редактировать
- История: возможность откатиться к предыдущей версии

- Рекурсивный обход выбранного фрейма

- Древовидный предпросмотр: старое имя → новое имя
- Чекбоксы для выборочного применения
- Цветовая индикация изменений
- Batch-применение с прогрессом

#### 3.4 Промпт для LLM-переименования

Текущая структура:
{{layerTree}}

## Структура UI

### Главный экран — табы:
1. **Генерация** — работа с текстом
2. **Данные** — управление локальными массивами
3. **Слои** — переименование
4. **Настройки** — провайдеры, пресеты, экспорт

### Компоненты
- `<ProviderSelector />` — выпадающий список провайдеров
- `<TokenDisplay tokens={n} cost={x} />` — показ токенов и стоимости
- `<StreamingText text={...} isStreaming={bool} />` — typewriter-эффект
- `<PreviewModal before={} after={} onApply onCancel />` — предпросмотр изменений
- `<PresetManager presets={[]} onSave onDelete />` — управление пресетами
- `<LayerTree layers={[]} changes={[]} />` — дерево слоёв с diff

## Обработка ошибок
- Таймаут запроса: показать сообщение + кнопка "Повторить"
- Невалидный API-ключ: подсветить поле, показать ошибку
- Превышение лимита токенов: предупреждение перед отправкой
- Пустой выбор: disabled кнопки + tooltip "Выберите текстовые слои"
- Сетевая ошибка: детальное сообщение + проверка CORS
