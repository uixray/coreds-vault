## 1. Концепция

Design Lint — модульный аудитор макета с тремя независимыми режимами: **типографика (Typography)**, **цвета (Colors)** и **эффекты (Effects)**. Плагин находит несвязанные с токенами/стилями значения, группирует их, позволяет массово переназначить — и при этом **не замораживает Figma** и **не трогает инстансы** без явного разрешения пользователя.

### Ключевая идея

> «Найди → Покажи → Переназначь» — три шага для наведения порядка в файле

Плагин работает как **обратный инспектор**: вместо просмотра свойств одного элемента — показывает ВСЕ уникальные значения в scope и сколько элементов их используют. Пользователь видит полную картину и принимает решение о массовой замене.

### 3.2 Режим Typography

#### 3.2.1 Анализ (Scan)

Для каждого TextNode в scope собираем:

| Свойство | API | Тип |
|---|---|---|
| Шрифт (font family + style) | `fontName` | FontName |
| Размер (font size) | `fontSize` | number |
| Межстрочный (line height) | `lineHeight` | LineHeight |
| Межбуквенный (letter spacing) | `letterSpacing` | LetterSpacing |
| Межабзацный (paragraph spacing) | `paragraphSpacing` | number |
| Вес (font weight) | `fontWeight` | number (readonly) |
| Привязка к TextStyle | `textStyleId` | string или `""` |
| Привязка к Variables | `boundVariables` | VariableAlias |
| TextStyle overrides | `getStyledTextSegments(['textStyleOverrides'])` | TextStyleOverrides[] |

**Обработка mixed values:**  
TextNode может иметь `figma.mixed` для `fontSize`, `fontName` и других свойств, если в одном текстовом блоке используются разные стили. В этом случае используем `getStyledTextSegments()` для получения всех сегментов с их индивидуальными свойствами.

**Группировка результатов:**

Результат сканирования — таблица уникальных комбинаций с количеством нод:

```

─────────────────────────────────────────────────────────────────────

Inter                Regular    16     24px     —                  23  ⚠
Inter                Bold       24     32px     Heading/H2        156
Roboto               Regular    14     20px     —                  12  ⚠
Inter                Medium     14     Auto     Caption/Medium     89
```

- `⚠` — не привязан к стилю/переменной (unlinked)
- Сортировка: по count desc (самые частые наверху)

### 3.5 Режим Override Detective

Кросс-режимный функционал: доступен как дополнительная вкладка или как переключатель внутри каждого режима при включенном флаге «Включить инстансы».

**Что ищет:**
- Инстансы, в которых переопределены fills/strokes (не совпадают с мастером)
- Инстансы, в которых переопределены text styles (через `textStyleOverrides`)
- Инстансы, в которых переопределены effects
- **НЕ** считает оверрайдом: текстовое содержимое (`characters`), видимость слоёв (`visible`), вложенные instance swap

**Структура результатов:**

```

  └ Sublayer "Container": shadow изменён

  └ Sublayer "Image": image fill изменён
```

**Действия:**
- **Сбросить конкретный тип оверрайда** — через прямую запись свойств из мастера (не `resetOverrides()`, который сбрасывает ВСЁ включая текст)
- **Сбросить все стилевые оверрайды** — fills + strokes + effects, но **сохранить** текст, видимость, instance swap
- **Показать** — выделить проблемные инстансы на канвасе

## 4. Критические технические решения

### 4.1 Chunked Processing — предотвращение зависания Figma

Figma плагины работают на main thread — любая длительная операция **замораживает весь UI Figma**. Официальная рекомендация Figma — разбивать работу на чанки с `setTimeout` между ними.

**Реализация:**

```typescript
// Размер чанка — количество нод за одну итерацию
const CHUNK_SIZE = 50;
// Пауза между чанками (мс) — даёт Figma обработать UI events
const YIELD_DELAY = 1;

// Утилита yield — отдаёт управление Figma main thread
function yieldToMain(): Promise<void> {
  return new Promise(resolve => setTimeout(resolve, YIELD_DELAY));
}

// Chunked обход дерева с промежуточной отправкой результатов
async function scanNodes(
  nodes: SceneNode[],
  processor: (node: SceneNode) => ScanResult | null,
  onChunkDone: (results: ScanResult[], progress: number) => void,
  signal: { cancelled: boolean }
): Promise<ScanResult[]> {
  const allResults: ScanResult[] = [];
  const totalNodes = nodes.length;
  
  for (let i = 0; i < totalNodes; i += CHUNK_SIZE) {
    // Проверяем отмену
    if (signal.cancelled) break;
    
    const chunk = nodes.slice(i, i + CHUNK_SIZE);
    const chunkResults: ScanResult[] = [];
    
    for (const node of chunk) {
      const result = processor(node);
      if (result) chunkResults.push(result);
    }
    
    allResults.push(...chunkResults);
    
    // Отправляем промежуточные результаты в UI
    const progress = Math.min((i + CHUNK_SIZE) / totalNodes, 1);
    onChunkDone(chunkResults, progress);
    
    // Yield — Figma может обработать клик на Cancel
    await yieldToMain();
  }
  
  return allResults;
}
```

**Стриминг результатов в UI:**

Результаты группировки пересчитываются при каждом чанке и отправляются в UI через `postMessage`. UI обновляет таблицу инкрементально — пользователь видит промежуточные результаты по мере сканирования.

```typescript
// code.ts — отправка промежуточных результатов
onChunkDone: (chunkResults, progress) => {
  // Добавляем новые результаты в общую группировку
  mergeIntoGroups(globalGroups, chunkResults);
  
  figma.ui.postMessage({
    type: 'scan-progress',
    progress: Math.round(progress * 100),
    groups: serializeGroups(globalGroups),
    scannedCount: processedNodes,
    totalCount: totalNodes,
  });
}
```

```javascript
// ui.html — инкрементальное обновление
window.onmessage = (event) => {
  const msg = event.data.pluginMessage;
  
  if (msg.type === 'scan-progress') {
    updateProgressBar(msg.progress);

  }

    enableActions();
  }
};
```

### 4.2 Сбор нод для сканирования (Flatten Tree)

Перед сканированием дерево нод разворачивается в плоский массив с учётом настроек:

```typescript
interface ScanOptions {
  includeHidden: boolean;    // включать node.visible === false
  includeInstances: boolean; // включать содержимое инстансов
}

async function collectNodes(
  root: SceneNode | PageNode,
  options: ScanOptions
): Promise<SceneNode[]> {
  const nodes: SceneNode[] = [];
  
  async function traverse(node: SceneNode, insideInstance: boolean) {
    // Пропуск скрытых (если не включены)
    if (!options.includeHidden && !node.visible) return;
    
    // Пропуск содержимого инстансов (если не включены)
    // Но сам Instance node — добавляем для подсчёта
    if (insideInstance && !options.includeInstances) return;
    
    nodes.push(node);
    
    if ('children' in node) {
      for (const child of node.children) {
        const childInsideInstance = insideInstance || node.type === 'INSTANCE';
        await traverse(child, childInsideInstance);
      }
    }
  }
  
  if ('children' in root) {
    for (const child of root.children) {
      await traverse(child, false);
    }
  } else {
    await traverse(root, false);
  }
  
  return nodes;
}
```

**Важно:** при `includeInstances = false` мы пропускаем **все дочерние ноды** внутри `INSTANCE`, но сам Instance node остаётся в списке. Это позволяет считать количество инстансов без обработки их содержимого.

### 4.3 Определение «настоящей видимости» (True Visibility)

Свойство `node.visible` — НЕ гарантия видимости. Нода может быть `visible: true`, но её родитель — `visible: false`. Для корректного определения нужна проверка всей цепочки:

```typescript
function isTrulyVisible(node: SceneNode): boolean {
  let current: BaseNode | null = node;
  while (current && current.type !== 'PAGE' && current.type !== 'DOCUMENT') {
    if ('visible' in current && !current.visible) return false;
    if ('opacity' in current && current.opacity === 0) return false;
    current = current.parent;
  }
  return true;
}
```

Для UI результатов: ноды из скрытых слоёв помечаются бейджом `👁 hidden` чтобы пользователь видел контекст.

### 4.4 Обнаружение стилевых оверрайдов в инстансах

Figma API предоставляет `InstanceNode.overrides` — массив всех полей с прямыми оверрайдами. Однако это **только прямые** оверрайды, без inherited.

Для детального анализа оверрайдов по типу (fills/strokes/effects) используем сравнение свойств sublayer-ов с соответствующими слоями мастер-компонента:

```typescript
interface StyleOverride {
  instanceId: string;
  instanceName: string;
  sublayerName: string;
  sublayerPath: string;       // "Button / Background"
  overrideType: 'fills' | 'strokes' | 'effects' | 'textStyle' | 'imageFill';
  currentValue: any;          // текущее значение в инстансе
  masterValue: any;           // значение из мастера
}

async function detectStyleOverrides(
  instance: InstanceNode
): Promise<StyleOverride[]> {
  const overrides: StyleOverride[] = [];
  const mainComponent = await instance.getMainComponentAsync();
  if (!mainComponent) return overrides;
  
  // Рекурсивно сравниваем sublayer-ы инстанса и мастера
  await compareChildren(instance, mainComponent, overrides, '');
  
  return overrides;
}

async function compareChildren(
  instanceParent: SceneNode & ChildrenMixin,
  masterParent: SceneNode & ChildrenMixin,
  overrides: StyleOverride[],
  path: string
) {
  for (let i = 0; i < instanceParent.children.length; i++) {
    const iChild = instanceParent.children[i];
    const mChild = masterParent.children[i];
    if (!mChild) continue;
    
    const childPath = path ? `${path} / ${iChild.name}` : iChild.name;
    
    // Проверяем fills
    if ('fills' in iChild && 'fills' in mChild) {
      const iFills = iChild.fills;
      const mFills = mChild.fills;
      
      if (iFills !== figma.mixed && mFills !== figma.mixed) {
        // Отдельная обработка для image fills
        const hasImageOverride = hasImageFillOverride(iFills, mFills);
        const hasColorOverride = hasColorFillOverride(iFills, mFills);
        
        if (hasImageOverride) {
          overrides.push({
            instanceId: (instanceParent as any).id,
            instanceName: (instanceParent as any).name,
            sublayerName: iChild.name,
            sublayerPath: childPath,
            overrideType: 'imageFill',
            currentValue: iFills,
            masterValue: mFills,
          });
        }
        
        if (hasColorOverride) {
          overrides.push({
            instanceId: (instanceParent as any).id,
            instanceName: (instanceParent as any).name,
            sublayerName: iChild.name,
            sublayerPath: childPath,
            overrideType: 'fills',
            currentValue: iFills,
            masterValue: mFills,
          });
        }
      }
    }
    
    // Аналогично для strokes, effects, textStyleId...
    
    // Рекурсия для вложенных контейнеров (не для вложенных инстансов)
    if ('children' in iChild && 'children' in mChild && iChild.type !== 'INSTANCE') {
      await compareChildren(iChild as any, mChild as any, overrides, childPath);
    }
  }
}
```

### 4.5 Селективный сброс оверрайдов (без потери контента)

`InstanceNode.resetOverrides()` / `InstanceNode.removeOverrides()` сбрасывает **ВСЁ** — включая текст и видимость. Нам нужен **селективный сброс** — только стилевые свойства:

```typescript
async function resetFillOverrides(
  instance: InstanceNode,
  sublayerIndex: number
): Promise<void> {
  const mainComponent = await instance.getMainComponentAsync();
  if (!mainComponent) return;
  
  const iChild = instance.children[sublayerIndex];
  const mChild = mainComponent.children[sublayerIndex];
  
  if ('fills' in iChild && 'fills' in mChild) {
    // Копируем fills из мастера
    const masterFills = JSON.parse(JSON.stringify(mChild.fills));
    (iChild as any).fills = masterFills;
  }
}

async function resetEffectOverrides(
  instance: InstanceNode,
  sublayerIndex: number
): Promise<void> {
  const mainComponent = await instance.getMainComponentAsync();
  if (!mainComponent) return;
  
  const iChild = instance.children[sublayerIndex];
  const mChild = mainComponent.children[sublayerIndex];
  
  if ('effects' in iChild && 'effects' in mChild) {
    const masterEffects = JSON.parse(JSON.stringify(mChild.effects));
    (iChild as any).effects = masterEffects;
  }
}

// Сброс ВСЕХ стилевых оверрайдов но СОХРАНЕНИЕ контента
async function resetAllStyleOverrides(instance: InstanceNode): Promise<void> {
  const mainComponent = await instance.getMainComponentAsync();
  if (!mainComponent) return;
  
  async function resetRecursive(iNode: SceneNode, mNode: SceneNode) {
    // Сбрасываем fills (кроме image fills если нужно)
    if ('fills' in iNode && 'fills' in mNode) {
      const mFills = mNode.fills;
      if (mFills !== figma.mixed) {
        (iNode as any).fills = JSON.parse(JSON.stringify(mFills));
      }
    }
    
    // Сбрасываем strokes
    if ('strokes' in iNode && 'strokes' in mNode) {
      (iNode as any).strokes = JSON.parse(JSON.stringify(mNode.strokes));
    }
    
    // Сбрасываем effects
    if ('effects' in iNode && 'effects' in mNode) {
      (iNode as any).effects = JSON.parse(JSON.stringify(mNode.effects));
    }
    
    // НЕ трогаем: characters, visible, componentProperties
    
    // Рекурсия (пропускаем вложенные инстансы)
    if ('children' in iNode && 'children' in mNode && iNode.type !== 'INSTANCE') {
      for (let i = 0; i < (iNode as any).children.length; i++) {
        const ic = (iNode as any).children[i];
        const mc = (mNode as any).children[i];
        if (mc) await resetRecursive(ic, mc);
      }
    }
  }
  
  await resetRecursive(instance, mainComponent);
}
```

### 4.6 Массовая замена с учётом скрытых нод

Проблема конкурентов: при массовой замене скрытые ноды пропускаются, и после их показа обнаруживаются «нетронутые» значения.

**Наше решение:** если при сканировании включен флаг «Включить скрытые слои», то замена тоже применяется к скрытым. Если флаг выключен — скрытые не сканируются и не заменяются. Логика простая: **что попало в результаты сканирования — то и заменяется**.

```typescript
// Каждый результат хранит ID ноды
interface GroupedResult {
  key: string;              // хеш группировки (hex+opacity+context)
  displayInfo: DisplayInfo; // данные для отображения
  nodeIds: string[];        // все ID нод в группе
  hiddenNodeIds: string[];  // подмножество — скрытые ноды
}

// При замене — работаем со всем списком nodeIds
async function applyReplacement(
  group: GroupedResult,
  action: ReplacementAction
): Promise<number> {
  let changed = 0;
  
  for (const nodeId of group.nodeIds) {
    const node = await figma.getNodeByIdAsync(nodeId);
    if (!node || node.removed) continue;
    
    // node.visible не влияет на доступ через Plugin API
    // мы спокойно меняем скрытые ноды
    await applyAction(node, action);
    changed++;
    
    // Yield каждые N нод
    if (changed % CHUNK_SIZE === 0) await yieldToMain();
  }
  
  return changed;
}
```

**Ключевой факт:** `node.visible = false` **не мешает** плагину читать и записывать свойства ноды. Скрытая нода полностью доступна через API. Ограничение только в `figma.skipInvisibleInstanceChildren` — мы его **не включаем** при сканировании с флагом «Включить скрытые».

### 4.7 Отмена сканирования

Пользователь может нажать «Отмена» во время сканирования. Реализация через shared signal:


## 9. Edge Cases и обработка ошибок

### 9.1 Пустое выделение

При scope = Selection и пустом выделении — показать сообщение: «Выделите элементы для сканирования или переключите scope на Page/Document».

### 9.2 Огромные файлы (100k+ нод)

- Chunked processing предотвращает зависание
- При Document scope — предупреждение: «Документ содержит ~N нод. Сканирование может занять некоторое время. Продолжить?»
- Прогресс-бар + кнопка отмены обязательны

### 9.3 figma.mixed

Свойства `fontSize`, `fontName`, `fills` могут возвращать `figma.mixed` при mixed values. Обязательная проверка:

```typescript
// Для TextNode
if (textNode.fontSize === figma.mixed) {
  // Используем getStyledTextSegments для посегментного анализа
  const segments = textNode.getStyledTextSegments([
    'fontSize', 'fontName', 'fontWeight', 'lineHeight',
    'letterSpacing', 'textStyleId', 'boundVariables'
  ]);
  // Каждый сегмент — отдельная запись
}
```

### 9.4 Remote (библиотечные) стили и компоненты

- `style.remote === true` — стиль из библиотеки, read-only
- `component.remote === true` — компонент из библиотеки
- При сбросе оверрайдов на инстансах библиотечных компонентов — `getMainComponentAsync()` может вернуть remote component. Свойства читаются, но сам компонент нельзя модифицировать. Для сброса — **копируем значения** из main component в инстанс.

### 9.5 Ноды удалены во время сканирования

Пользователь может удалить ноду пока плагин работает. Проверка `node.removed` перед любой операцией:

```typescript
const node = await figma.getNodeByIdAsync(nodeId);
if (!node || node.removed) {
  // пропускаем, не крашимся
  continue;
}
```

### 9.6 Шрифты не загружены при замене текстовых свойств

Перед изменением любых текстовых свойств — обязательная загрузка шрифтов. Для mixed-font TextNode — загрузить ВСЕ шрифты:

```typescript
async function loadAllFontsForTextNode(node: TextNode): Promise<void> {
  if (node.fontName === figma.mixed) {
    const segments = node.getStyledTextSegments(['fontName']);
    const fonts = new Set(segments.map(s => JSON.stringify(s.fontName)));
    for (const fontStr of fonts) {
      await figma.loadFontAsync(JSON.parse(fontStr));
    }
  } else {
    await figma.loadFontAsync(node.fontName);
  }
}
```

### 9.7 Gradient fills

Градиенты группируются по типу (`GRADIENT_LINEAR`, `GRADIENT_RADIAL`, `GRADIENT_ANGULAR`, `GRADIENT_DIAMOND`) с превью. Массовая замена для градиентов поддерживается через полную замену paint объекта.
